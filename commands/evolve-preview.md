# /evolve-preview

Brief: User-facing command to manage preview servers for HTML/CSS/JS mockups generated by Evolve. Spins up a lightweight local server bound to `127.0.0.1`, returns a viewable URL, and tracks running servers so they can be listed or killed cleanly.

## Usage

| Form | Behavior |
| --- | --- |
| `/evolve-preview` | No args — list all currently running preview servers (port, directory, PID, uptime). |
| `/evolve-preview <dir>` | Start a preview server for `<dir>` on the next available port. |
| `/evolve-preview <dir> --port N` | Start a preview server for `<dir>` on a specific port `N`. |
| `/evolve-preview --kill <port>` | Kill the preview server bound to `<port>`. |
| `/evolve-preview --kill-all` | Kill every running preview server tracked by this session. |

## What I do when invoked

1. Parse the user's args into one of the five usage forms above.
2. Map the parsed form to flags for `scripts/preview-server.mjs`:
   - no args -> `node scripts/preview-server.mjs --list`
   - `<dir>` -> `node scripts/preview-server.mjs --serve <dir>`
   - `<dir> --port N` -> `node scripts/preview-server.mjs --serve <dir> --port N`
   - `--kill <port>` -> `node scripts/preview-server.mjs --kill <port>`
   - `--kill-all` -> `node scripts/preview-server.mjs --kill-all`
3. Run the script as a child process and capture stdout/stderr.
4. Format the output for the user:
   - On start: surface the URL (`http://localhost:PORT`), bound directory, and PID.
   - On list: render a compact table.
   - On kill: confirm which servers were stopped.
   - On error: print the exact error and a one-line hint (e.g. port already in use -> suggest `--port` with a free port).

## Notes

- Auto-cleanup on session end: tracked PIDs are terminated when the Claude Code session exits, so leftover servers don't pile up.
- Hot-reload by default: file changes inside the served directory trigger a browser refresh; pass `--no-reload` to disable.
- `127.0.0.1` only: the server binds to loopback (never `0.0.0.0`), so previews are not reachable from other machines on the network.
