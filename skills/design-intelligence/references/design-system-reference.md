# Design System Reference

Design-system guidance is used to preserve approved tokens and component contracts.

Rules:
- Treat `.supervibe/artifacts/prototypes/_design-system/manifest.json` with `status: approved` as source of truth.
- New colors, typography, motion, spacing, or components require a narrow extension request.
- Validate token drift before delivery.
- Component states must include resting, hover, active, focus, disabled, loading, empty, and error when applicable.
