# Asset And Collateral Reference

Collateral data covers logo, icon, CIP deliverable, mockup context, and social/banner guidance.

Rules:
- Asset output format, size, and usage context must be explicit.
- Palette and type choices must map to approved brand tokens.
- Mockup context should serve the product message rather than become decorative noise.
- Treat generated asset prompts as drafts until reviewed against brand and accessibility constraints.
